Key Design Decisions

Strategy Pattern — Each channel implements NotificationChannelHandler interface. New channels auto-register via Spring component scan.
Exponential Backoff — Failed notifications retry with delay = initialDelay × 2^retryCount. Max 3 attempts before permanent failure.
Scheduled Processing — Spring @Scheduled jobs poll for due scheduled, recurring, and retryable notifications at fixed intervals.
Priority Queue — Notifications ordered by priority (HIGH → MEDIUM → LOW) at the database query level before processing.
Batch Processing — Bulk API partitions large user lists into configurable batch sizes for efficient campaign delivery.
Separation of Concerns — Controllers → Services → Channel Handlers → Repositories. Each layer has a single responsibility.
